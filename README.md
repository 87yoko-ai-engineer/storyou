# Storyou（ストーリュー）

> **Story + You** — あなたが主人公の、ストーリー型AI英会話アプリ。

AI英会話アプリ「スピークバディ」に着想を得て、**ストーリーのセリフを声に出し、発音を採点してもらいながら会話練習できる**スマホ向けWebアプリです。「録音 → 文字起こし → 採点(A/B/C) → キャラクターの反応が分岐」という体験を、**拡張しても壊れない設計**で実装しています。

🔗 **デモ**：_（Vercelデプロイ後にURLを記載）_

---

## ✨ 主な機能（MVP）

- ストーリーに沿ってキャラクターが英語で話す（音声合成）
- 主人公（＝あなた）のセリフを**録音 → 文字起こし**
- 発音の聞き取りやすさを**採点し、A / B / C で判定**
- 判定に応じて**キャラの反応が分岐**（Good job! など）
- 1ストーリーを通すと**クリア**

> 現状は最小版（MVP）です。採点は「文字起こし結果とお手本の一致率」による簡易版で、本格的な発音評価（音素レベル）は今後の差し替えで対応予定です（下記ロードマップ）。

## 🛠 技術スタック

- **Next.js 16**（App Router） / **React 19** / **TypeScript**
- **Tailwind CSS 4**
- 音声：ブラウザ標準 API（Web Speech API / SpeechSynthesis）— MVPはバックエンド不要
- デプロイ：**Vercel**

## 🧩 設計のポイント（このプロジェクトの肝）

「後から機能を足しても壊れない」ことを最優先に、責務を**4つの層**へ分離しています。

| 層 | 役割 | 性質 |
|---|---|---|
| Content | ストーリー・セリフ等のデータ | 増える（コードではなくデータ） |
| Engine | 再生の状態機械（録音→採点→分岐） | ほぼ固定 |
| Adapter | 録音・音声合成・採点の「差し込み口」 | 中身は差し替え可・形は固定 |
| UI | 画面 | プラットフォーム依存 |

- **ストーリーはコードではなくデータ** → 追加は `content/stories/` にファイルを足すだけ。
- **アダプタは差し込み口だけ固定** → 音声認識や採点のAPIを差し替えても Engine は無傷。
- 詳細は設計ドキュメント参照：
  - [docs/01-requirements.md](docs/01-requirements.md)（要件定義）
  - [docs/02-design.md](docs/02-design.md)（設計：データモデル・状態機械・アダプタ・拡張マップ）

## 🚀 ローカルで動かす

> 音声認識は **Google Chrome / Edge** での利用を推奨（Firefox は Web Speech API 非対応）。

```bash
npm install
npm run dev
# http://localhost:3000 を開く
```

その他のコマンド：

```bash
npm run build      # 本番ビルド
npx tsc --noEmit   # 型チェック
```
