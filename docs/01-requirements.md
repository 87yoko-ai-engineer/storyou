## 1. プロジェクト概要

### 一言でいうと
**ストーリーに沿って英語のセリフを声に出し、発音を採点してもらいながら会話練習できる、スマホ向けのAI英会話アプリ。**

### アプリ名
**Storyou（ストーリュー）** … "Story（物語）＋ You（あなた）" の造語。**「あなたが主人公の物語」** という意味を込めている。日本語話者にも意味が伝わりやすい点も決め手。

### 背景・動機
- AI英会話アプリ「スピークバディ」を実際に使って良いと感じ、**同じような仕組みを自分で作れるか試したい**。
- **ポートフォリオ**として、技術力と「設計を考えられること」を見せたい。
- 完成形は**スマホで動くアプリ**。開発中はPCのブラウザで動かす（同じコードがそのままスマホでも動く形にする）。

### ゴール
- まず **1ストーリー** を最後まで遊べる最小版（MVP）を完成させる。
- そこに機能を**足していくだけ**で本家に近づけられる、壊れない土台をつくる。

---

## 2. 想定ユーザー

| 対象 | 説明 |
|---|---|
| 使う人 | 英語を「声に出して」練習したい人。人間相手だと緊張する／予約が面倒、という層。 |
| 見せる相手 | （ポートフォリオとして）採用担当者・エンジニアなど。URLを開いてすぐ試せることを重視。 |

---

## 3. コアコンセプト（体験の核）

> ストーリーの中で**主人公のセリフを自分で録音** → **発音を聞き取りやすさで採点** → **A / B / C の判定でキャラクターの反応が変わる** → セリフを進めて**ストーリーをクリア**。

この「録音 → 採点 → 反応が分岐」の手応えが、このアプリのいちばんの面白さ。

---

## 4. スコープ（どこまで作るか）

### ✅ MVP＝最初に作る範囲（= 1ストーリーを通しで遊べる）
1. 1つのストーリーを開始できる
2. AIキャラクターのセリフが**文字＋音声**で再生される
3. 主人公（＝ユーザー）の番で**録音**できる
4. 録音を**文字起こし**し、**発音の聞き取りやすさを採点**する
5. 採点結果を **A / B / C** に判定し、**キャラの反応を分岐**させる
   - A: 「Good job!」
   - B: 「よく頑張ってるね」
   - C: 「もうちょっと頑張ろう」
6. セリフを順に進め、最後まで行くと**「ストーリークリア」**が表示される

### 🚫 今は作らない（Out of Scope：あえて外す）
- ログイン／会員登録
- 課金・サブスク
- 複数の章・複数ストーリーの管理（MVPは1ストーリーのみ）
- キャラクターの訛り再現（インドネシア系・中国系など）→ まずは**ネイティブ発音1種**
- ストーリーを画面から編集する管理ツール
- 細かい発音の弱点分析・復習機能

> 「外すもの」をはっきり書くのは、迷いと作りすぎを防ぐため。これらは下の「将来の拡張」で順に足していく。

### 🔭 将来の拡張（Backlog：足していく順番の目安）
- **Phase 2**: 字幕の和訳・言い換え提案
- **Phase 3**: 本格的な発音評価（音素レベルの採点）に差し替え
- **Phase 4**: 章・複数ストーリー・進捗保存（章クリア）
- **Phase 5**: 訛りつき音声、ログイン、復習機能、（必要なら）アプリストア配信
- **ストーリーのストック（題材候補）**: A2（請求書差異の説明）／ A3（監査対応）／ C1（コードレビュー）／ C2（バグ報告）／ C3（実装方針の相談）。ストーリー追加時に使う。

## 5. MVPの主要機能（もう少し具体的に）

| # | 機能 | やること |
|---|---|---|
| F1 | ストーリー再生 | セリフを順番に表示・進行する |
| F2 | キャラの発話 | AIのセリフを文字表示＋音声で再生する |
| F3 | 録音 | ユーザーの番でマイク録音する |
| F4 | 文字起こし（ASR） | 録音音声をテキストに変換する |
| F5 | 採点 | お手本テキストとどれだけ一致するかで点数を出す（MVPは簡易版） |
| F6 | 判定と分岐 | 点数を A/B/C に分け、対応するキャラの反応を出す |
| F7 | クリア表示 | 全セリフ終了で「ストーリークリア」を出す |

> **採点について（MVPの方針）**: 最初は「文字起こし結果がお手本にどれだけ近いか（一致率）」で点数を出す簡易版にする。追加費用なしで作れて、A/B/C分岐を動かすには十分。本格的な発音評価は後から**差し替える**（土台は壊さない）。

---

## 6. 体験フロー（1ストーリーの流れ）

```
1. ストーリーを開始
2. AIキャラがセリフを話す（文字＋音声）
3. 「あなたの番」→ お手本セリフが表示される
4. ユーザーが録音して話す
5. 文字起こし → 採点 → A/B/C 判定
6. 判定に応じてキャラが反応（褒める／励ます）
7. 次のセリフへ（2〜6を繰り返す）
8. 全部終わったら「ストーリークリア！」
```

## 6-α. 最初に作るストーリー（確定）— A1：月次決算の進捗報告

- 場面：朝の短いチェックイン（オンライン通話）
- 登場人物：**あなた**（経理担当・主人公）／ **Sarah**（海外オフィスのマネージャー）
- 録音ポイント：3か所（#2・#4・#6）

| # | 話す人 | 英語のセリフ | 和訳 |
|---|---|---|---|
| 1 | 🔊 Sarah | Good morning! How's the monthly closing going? | おはよう！月次決算の進み具合はどう？ |
| 2 | 🎤 あなた（録音） | Good morning, Sarah. It's going well. We've finished about eighty percent. | おはようございます、サラ。順調です。8割ほど終わりました。 |
| 3 | 🔊 Sarah | Great. When do you think you can finish the rest? | いいね。残りはいつ終わりそう？ |
| 4 | 🎤 あなた（録音） | I expect to finish by Thursday this week. | 今週の木曜までには終わる見込みです。 |
| 5 | 🔊 Sarah | Perfect. Is there anything you need help with? | 完璧。何か手伝えることはある？ |
| 6 | 🎤 あなた（録音） | Not for now. I'll let you know if anything comes up. | 今のところ大丈夫です。何かあれば連絡します。 |
| 7 | 🔊 Sarah | Sounds good. Thanks for the update! | いいね。報告ありがとう！ |

> この確定セリフが、設計後に「コンテンツ第1号データ」になる。

## 7. 技術方針（現時点の決定事項）

| 項目 | 決めたこと | 理由 |
|---|---|---|
| 形態 | **スマホ向けWebアプリ**（モバイルファースト） | ゴールはスマホ。Webなら同じコードがPC（開発用）でもスマホでも動く |
| フレームワーク | **Next.js** を想定 | 開発・デプロイがしやすく、URLで共有＝ポートフォリオ向き |
| 配信（将来） | PWA化 → 必要ならCapacitorでアプリ化 | 「後から足す」で対応でき、土台を壊さない |
| AI処理 | ASR・LLM・音声合成・採点は**クラウドAPI**を利用 | スマホの性能に依存せず、機能が劣化しない |
| 差し替え | ASR/音声合成/採点は**差し込み口（インターフェース）経由**で使う | 後でAPIを差し替えても本体（エンジン）を触らずに済む |

> 具体的にどのAPI（どの会社のサービス）を使うかは「未決事項」。設計段階で比較して決める。

## 8. 非機能要件（軽く）

- **反応速度**: 録音後、採点と反応までの待ち時間が長すぎないこと（沈黙が続くと体験が悪い）。
- **コスト**: APIは使った分だけ課金。試作中に使いすぎない範囲で。
- **対応端末**: スマホのブラウザで動くこと（開発中はPCブラウザ）。
- **マイク**: ブラウザのマイク機能を使う（iOSのSafariは少し癖がある点に留意）。

---

## 9. 成功基準（MVPの「完成」とは）

次がすべて満たせたらMVP完成とする：

- [ ] 1ストーリーを最初から最後まで進められる
- [ ] AIのセリフが文字＋音声で出る
- [ ] 自分の声を録音して文字起こしできる
- [ ] 採点結果が A/B/C で出て、キャラの反応が変わる
- [ ] 最後に「ストーリークリア」が表示される

## 付録：用語ミニ辞典（初心者向け）

| 用語 | 意味 |
|---|---|
| MVP | Minimum Viable Product。「ちゃんと動く最小の製品」。まず作る範囲のこと。 |
| ASR | Automatic Speech Recognition。音声をテキストに変換する技術（文字起こし）。 |
| 音声合成（TTS） | Text To Speech。テキストを音声（声）に変換する技術。 |
| LLM | 大規模言語モデル。返答を考えるAI（ChatGPTやClaudeなど）。 |
| Web/PWA | ブラウザで動くアプリ。PWAにするとスマホのホーム画面にアプリとして入れられる。 |
| アダプタ／インターフェース | 「差し込み口」。中身（使うAPI）を後で差し替えても、本体を触らずに済む仕組み。 |
